# ShaiHive

AI翻译+人工审阅版

## 由可验证凭证 (VCs) 驱动的数字发展！
ShaiHive 本质上，是为了所有希望能够掌控代笔、掌控链上资产来打造得一个Shaicoin得移动钱包。Shaihive将Neutrino轻客户端与可验证凭证（VC）钱包相结合，使用户能够在单一体验中持有密钥、余额和一些数字资产（为了便于理解，比如游戏中得一些皮肤、道具，就可以看作一种数字资产）。

这个Shaihive钱包是迈向 **Gaming Requests for Comment（GRC）**得第一步。（这个不太好翻译，保留原味！） GRC 是一种开放规范（我们就保留这个原汁原味的叫法！），旨在让你的装备、道具和资产在不同的游戏、漫展和元宇宙之间自由穿梭，且所有权**真实可证**。

## 我们为什么要折腾GRC？

现在的数字资产简直就是“坐牢”。 不管是我们得皮肤、头像、收藏品、成就……这些实际上都被锁在创造它们的平台中。一旦游戏工作室关服或者改了霸王条款（emmm比如CSGO？），你的所有权瞬间归零。 身份认证是依靠中心化数据库得，资产转移那就做梦（至少企鹅不会给你转移），更没有一个中立的注册表来证明“是谁发行了这玩意儿”。

所以，我们在赛博世界里，理应享有和现实世界一样的权利：**真正拥有、自由展示、随身携带**。

## ShaiHive让梦想照进现实的核心功能

- **Neutrino 轻钱包 (真·去中心化)**： 直接通过紧凑过滤器 (compact filters) 连接 Shaicoin 全节点。区块头、过滤器、UTXO 统统缓存在你的手机上。**拒绝托管，拒绝信任第三方**，你的余额和交易，你自己说了算。
- **一把钥匙，走遍天下 (确定性密钥 + DID 根)**： 一套助记词搞定所有。支持legacy, SegWit, Taproot等多种格式地址。最酷的是，你的支付密钥同时也用来签署 VC (可验证凭证)。**把“身份”和“资产”死死绑定在同一把私钥上。**（别急，我解释一下，DID就是去中心化得数字身份）
- **凭证指挥中心**： `CredentialsScreen` 不只是个列表，它是你的数字卡包。凭证被渲染成可交互的“集换式卡牌”。你可以铸造演示凭证，点击翻转卡牌查看背后的 JSON 证明，并在本地模拟展示、转移或分享。
- **支付与资产，无缝合体**： 发送/接收界面、UTXO 管理、费率滑块、内存池验证……这些硬核金融功能与凭证发行功能并以此列。因为在我们看来，**经济行为和数字资产行为，本就该共享同一个信任层。**
- **极客控制台 (Operator Controls)**： 专为高级玩家准备。简单来说就是我们可以把钱包指向我们信任的 Shaicoin 节点，同时管理黑/白名单节点，从指定高度重扫区块，甚至重置本地状态。未来，GRC 索引器和发行者注册表也将接入这里。

## 钱包里的 VC 玩法

VC也就是凭证！可验证数字凭证！

1. **铸造 (Mint)**：App 从你的 SegWit 地址派生出一个 DID，用 secp256k1 算法签名声明，然后在本地生成并存储带有证明元数据的 VC。

2. **鉴赏 (Inspect)**：正面是炫酷的全息卡牌，展示立绘、背景故事和属性；背面则是给开发者或验证者看的 JSON 代码。

3. **展示 (Present)**：模拟创建一个“可验证展示” (Verifiable Presentation)。想象一下，你在电竞比赛、交易大厅或 AR 入口处，掏出手机“刷”一下卡。这个卡实际上适合你的VC有一定关系得，可以理解为卡上有你得身份证号（虽然这样不对，但是可以理解为这里得可验证展示VP和VC具有一定得密码学上得验证关联。）

4. **转移或分享 (Coming Soon)**：UI 已经预留了位置。未来，你可以转移VC得所有权，或者把VC打包，在不同的 App 之间进行交换。

   

## 从钱包到 GRC：下一盘大棋

GRC 是将 ShaiHive 正在原型化的东西**标准化**：

- **可验证数据注册表 (VDR)**：Shaicoin 交易记录发行者的公钥。任何人都可以通过区块链或索引器，来查询一个资产是不是链上发行的正版资产。
- **链下资产，链上信任**：为了速度，资产数据不上链；只有**所有权转移**或**撤销**才上链。钱包现在的做法就是这样：VC 载荷存在手机本地，但资金安全依靠 Shaicoin 共识。
- **信任铁三角**：发行者铭刻密钥到持有者在 ShaiHive 中保管凭证到验证者查询 VDR。钱包现在的流程，就是在教用户习惯这种“一切尽在掌握”的感觉。
- **标准化数据类型**：GRC 定义了各种模式（头像、武器、音效包、材质、UI 套件、虚拟世界、成就）。这确保了你在 ShaiHive 里看到的 VC，扔进 Unity、Unreal 或自研引擎里，照样能跑。

## 场景模拟：线下卡牌的交换

1. **发行**：卡牌发行商在 VDR 上注册 Shaicoin 公钥，铸造带有背景故事、稀有度和编号的 VC 卡牌。
2. **领取**：收藏家下载 ShaiHive，同步节点，接收代表卡牌的凭证。
3. **验货**：在线下或虚拟展会上，展位运行着基于 GRC SDK 的验证程序。收藏家出示钱包里的凭证，**扫码即验真伪，且不泄露隐私**。
4. **交换**：两个收藏家互换资产。ShaiHive 签发一个新的凭证，引用发行者的密钥来更新所有权。**只有所有权的变更会被记录在链上**。
5. **跨界**：随后，这张卡牌出现在了一个支持 GRC 的游戏中。游戏 SDK 读取 VC 载荷，加载基础属性，并根据游戏规则添加特效。

## 为什么这是跨世界资产的基石？

- **资产“润”得丝滑**：ShaiHive 现在就能导出 JSON；未来直接通过同样的证明机制，把资产流式传输到游戏、AR 层和市场中。
- **安全飞地**：代币、DID、凭证，全在一个安全屋里。无论是在展会还是线上，交易都在你私钥的掌控之下。
- **游戏互通性**：GRC 的分层元数据保证核心属性不变，同时允许每个游戏环境添加自己的修改器、皮肤或物理特性。
- **市场 + 变现**：Shaicoin 费用保障发行者注册；VC 市场可以用链上支付结算，VC 证明保障货真价实。
- **硬核安全**：VDR 的不可篡改性、加密展示、灵活的撤销/轮转流程——这一切都源自钱包已经在运行的底层原语。



## 编译安装我们自己的钱包（基于flutter）
1. **安装并获取依赖**
   ```
   flutter pub get
   ```
2. **在目标设备上运行 ShaiHive**
   - Android 
     ```
     flutter run -d android
     ```
   - iOS 
     ```
     flutter run -d ios
     ```
   - macOS 
     ```
     flutter run -d macos
     ```
   - Windows 
     ```
     flutter run -d windows
     ```
3. **配置节点连接**
   在应用内，打开 **Settings → Node Settings** 并指向你的 Shaicoin 节点（默认为 P2P `42069`）。
4. **初始化钱包**
   创建或导入助记词，进行备份，并让 Neutrino 客户端同步区块头和过滤器。
5. **铸造并检查凭证**
   导航至 **Credentials → Demo** 发行 VC，查看其全息卡片，并翻转查看原始 JSON 证明。
6. **交易与验证**
   使用 **Receive** 获取新地址，使用 **Send** 广播交易，然后导出凭证 JSON 并通过原型 GRC 工具或 VDR 索引器运行它，以模拟跨世界导入。

## GRC 路线图
1. **VDR 协议 + 索引器**：最终确定铭刻格式和公共 API，以便验证者可以信任发行者数据。
2. **SDK 交付**：为 Unity、Unreal 和自定义引擎发布解析器、适配器和验证助手。
3. **钱包演进**：直接在 ShaiHive 内部添加凭证转移、市场挂钩和发行者发现功能。
4. **试点项目**：与工作室、卡牌发行者和活动组织者合作，进行现场互操作性演练。

## 未来：ShaiHive 奔赴星辰大海
ShaiHive 将成为跨世界资产交互的底层驱动。代币、游戏道具、头像、成就、活动门票和集换式卡牌将存放在由 Shaicoin 共识支持的一个去中心化账本中。用户将走进游戏或元宇宙中心，直接从钱包中出示证明。开发者将依靠 SDK 来认可这些证明，而无需重新发明安全性。

加入我们，让我们将主权所有权变成每位玩家、创作者、收藏者和建设者的默认体验。Shaicoin 是数字复兴的栖息地。
